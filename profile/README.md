# 데이터 전문 거래 플랫폼 Datcha
> 남은 데이터를 그냥 버리시나요?     
> 이제는 안전하게 가치로 바꾸세요.      
> 전화번호 없이도, 사기 걱정 없이도, 복잡한 절차 없이도 실시간 시세 정보를 바탕으로 공정하고 투명한 거래가 가능합니다.      
> 모바일 데이터가 안전한 디지털 자산이 되는 세상       
> 그 첫 걸음을 전문 거래 플랫폼 Datcha와 함께 시작하세요.

## 프로젝트 개요

### 해결하고자 하는 문제
**"비공식적이고 불안전한 모바일 데이터 거래 환경을 개선하여 안전하고 효율적인 전용 거래 플랫폼 구축"**

#### 현재 데이터 거래의 문제점
- **개인정보 노출 위험**: 전화번호 등 개인정보 교환 필수
- **거래 안전성 부족**: 오전송, 사기 등의 위험
- **시세 정보 부재**: 적정 거래 가격 판단 어려움
- **복잡한 거래 절차**: 중고거래 플랫폼 → 통신사 선물하기 → 개별 처리

#### 우리의 해결책
- **개인정보 보호**: 플랫폼 내 자체 시스템으로 안전한 거래
- **페이 시스템**: 서비스 내 재화를 통한 안전한 결제
- **실시간 시세**: 1시간마다 갱신되는 시장 가격 정보
- **원스톱 거래**: 거래부터 데이터 충전까지 한 번에

## 주요 가정사항
### 내부 시스템
- 사용자 데이터 현황, 플랫폼에서 자체 관리
- 통신사별 데이터 충전권 시스템 존재
- 통신사 간 데이터 거래 제한 (향후 확장 예정)

### 결제 시스템
- **토스 페이먼츠**: 간편결제 및 카드 결제
- **페이 관리**: 플랫폼 내 1:1 비율 재화 시스템
- **수수료 정책**: 환전 시 약 3% 수수료

## 핵심 기능

### 시스템 내부 재화 단위

| 재화 단위 | 단위 | 설명 |
|-----------|------|------|
| **소유 데이터** | MB/GB | 사용자가 보유한 사용가능한 데이터 (1000MB 초과시 GB로 표현) |
| **구매 데이터** | MB/GB | 데이터 충전권을 통해 충전된 데이터 (판매 가능 데이터로 전환 불가) |
| **판매 가능 데이터** | MB/GB | 소유 데이터에서 판매용으로 변환한 데이터 양 (거래에 사용됨) |
| **페이** | 원 | 데이터 거래 시 사용할 결제 단위 (현금과 1:1 비율로 충전 및 환전) |
| **데이터 충전권** | MB/GB | 거래 완료 시 발급되는 일련의 코드 (데이터 양 포함) |
| **이벤트 할인 쿠폰** | - | 이벤트 배너에서 발급되는 할인 코드 및 쿠폰 (페이 충전 시 사용 가능) |

### 데이터 관리 구조
```
[소유 데이터] → [판매 가능 데이터] → [거래] → [데이터 충전권] → [구매 데이터]
```

- **데이터 분리 관리**: 소유/판매 가능/구매 데이터 독립적 운영
- **100MB 단위 변환**: 판매 가능 데이터 최소 단위
- **개인별 초기화**: 가입일 기준으로 매월 데이터 초기화
- **일방향 변환**: 구매 데이터 → 판매 가능 데이터 변환 불가

### 안전한 거래 시스템
- **개인정보 보호**: 전화번호 등 개인정보 교환 없이 거래
- **사기 방지**: 플랫폼 내 자체 시스템으로 중간자 공격 방지
- **페이 시스템**: 1:1 비율의 서비스 내 재화로 안전한 결제
- **거래 보호**: 동시 거래 방지 및 선착순 원칙 적용

### 실시간 시세 정보
- **시세 그래프**: 1시간마다 갱신되는 실시간 시세 정보
- **통신사별 분석**: 각 통신사별 1GB 기준 거래 금액 제공
- **가격 비교**: 현재 시세 대비 거래 가격 비교 가능
- **합리적 의사결정**: 데이터 기반 거래 판단 지원

### 다양한 거래 방식
- **일반 판매**: 판매자 지정 가격으로 즉시 거래
- **입찰 판매**: 1시간 경매를 통한 시장 가격 형성
- **같은 통신사 거래**: 통신사별 호환성 보장

### 예상 거래 프로세스
```
1. 회원가입 → 요금제 자동 배정
2. 판매 가능 데이터 설정 → 거래용 데이터 분리
3. 거래 진행 → 일반 판매 또는 입찰 판매
4. 데이터 충전권 발급 → 구매자에게 충전권 제공
5. 충전권 등록 → 실제 사용 가능한 데이터로 전환
```

## 사용자 가이드

### 판매자 시나리오
**목표**: 보유 데이터를 판매하여 페이 획득 및 현금화

1. **회원가입 및 설정**
   - 이메일 인증 회원가입
   - 요금제 자동 배정 (예: 100GB)
   - 판매 가능 데이터 설정 (예: 20GB)

2. **판매글 작성**
   - 일반 판매: 5GB → 페이 5,000원 (사용자 지정)
   - 입찰 판매: 1GB → 페이 1,000원부터 시작

3. **거래 완료 및 정산**
   - 거래 완료 시 페이 지급
   - 구매자에게 데이터 충전권 발급
   - 페이 현금 환전 (1:1 비율)

### 구매자 시나리오
**목표**: 필요한 데이터를 안전하게 구매하여 사용

1. **회원가입 및 페이 충전**
   - 이메일 인증 회원가입
   - 페이 충전 (예: 10,000원 → 페이 10,000원)

2. **상품 탐색 및 구매**
   - 실시간 시세 확인
   - 일반 구매 또는 입찰 참여
   - 보유 페이 범위 내에서 구매

3. **데이터 사용**
   - 데이터 충전권 발급 (1일 내 등록 필요)
   - 충전권 등록 → 구매 데이터로 전환
   - 실제 데이터 사용

### 관리자 시나리오
**목표**: 플랫폼 운영 및 사용자 관리

1. **대시보드 관리**
   - 실시간 시세 모니터링
   - 회원 현황 및 거래 통계
   - 신고 접수 현황

2. **콘텐츠 관리**
   - 신고 내역 처리 (AI 1차 + 관리자 2차)
   - 제재 대상 판단 및 조치

## 운영 정책

### 회원 정책
- **가입 방식**: 이메일 인증 (전화번호 인증 불필요)
- **요금제 배정**: 가입 시 랜덤 요금제 자동 배정
- **데이터 초기화**: 개인별 가입일 기준 월별 초기화
- **닉네임**: 랜덤 제공 후 마이페이지에서 변경 가능

### 거래 정책
- **거래 조건**: 같은 통신사 사용자 간에만 거래 가능
- **판매 범위**: 판매 가능 데이터 내에서만 거래
- **게시글 유효기간**: 데이터 초기화 전날까지
- **구매 조건**: 보유 페이 이상 금액만 구매 가능

### 결제 정책
- **페이 시스템**: 현금과 1:1 비율
- **결제 수단**: 토스 페이먼츠 (간편결제, 신용/체크카드 등)
- **충전권 유효기간**: 발급 후 24시간 이내 등록 필수
- **환전**: 언제든지 1:1 비율로 현금 환전 가능 (수수료 별도)

### 안전 정책
#### 신고 및 제재 시스템
- **1단계 검토**: AI 자동 검수
- **2단계 검토**: 관리자 수동 검토
- **3단계 검토**: 관리자 제재 처리

#### 제재 기준
| 위반 사항 | 신고 누적 | 제재 내용 | 제재 기간 |
|-----------|-----------|-----------|-----------|
| 욕설 및 비속어 | 5회 | 게시글 작성 제한 | 7일 |
| 주제 불일치 | 5회 | 게시글 작성 제한 | 1일 |
| 음란 내용 | 즉시 | 영구 정지 | 영구 |
| 허위 신고 | 3회 | 신고 제한 | 30일 |
